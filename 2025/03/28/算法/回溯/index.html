把backtracking当成下一层问题，我们不关心，只关心本层问题，本层只有两个操作，add和remove。因此，最终path依然会回到初始状态。

不过有个例外，就是最后一层，这时候在调用回溯之前程序已经终止，无法调用回溯了，所以这时候path并不是初始状态。

关于为什么要新开一个ArrayList存放单一结果？

以39.组合总和为例

在 ans.add(new ArrayList<>(path)); 这一行中，为了确保每个组合的副本都被正确地存储在结果列表 ans 中，你创建了 path 的一个副本，并将其添加到 ans 中。

如果直接将 path 添加到 ans 中，那么在后续的回溯过程中，当你修改 path 时，已经添加到 ans 中的组合也会受到影响，因为它们共享同一个 path 实例。这样会导致 ans 中存储的组合不正确。

通过使用 new ArrayList<>(path) 创建 path 的副本，你确保了在修改 path 时不会影响到已经添加到 ans 中的组合。这样可以保证 ans 中存储的每个组合都是在不同的路径状态下生成的，从而保证结果的正确性。

这里涉及到浅拷贝与深拷贝的关系

  1. 浅拷贝：
    • 浅拷贝仅复制了对象本身，而不会复制对象内部包含的引用对象。也就是说，原始对象和其副本共享内部对象的引用。
    • 在浅拷贝中，如果修改了副本中的引用对象，那么原始对象中的对应对象也会受到影响。
  2. 深拷贝：
    • 深拷贝会复制对象本身以及对象内部所有的引用对象，而不仅仅是引用。也就是说，原始对象和其副本完全独立，对副本的修改不会影响原始对象。
    • 深拷贝通常需要递归地复制对象及其内部引用的对象，确保所有层级的对象都被完整地复制。
